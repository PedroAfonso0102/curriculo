# Regras do Projeto — curriculo

Este documento descreve regras acionáveis que devem ser seguidas ao desenvolver neste repositório.

## 1. Design e Cores
- Evitar `#000000` e `#FFFFFF`. Use tons próximos conforme o Design System.
- Superfície (Light): `--bg-surface: #FAFAFA`.
- Superfície (Dark): `--bg-surface: #121212` no tema escuro.
- Fundo da aplicação: `--bg-app: #F2F2F7`.
- Hairline: `--border-hairline: #E5E5EA`.

Exemplo mínimo de declaração de variáveis (CSS):
```
:root {
  --bg-app: #F2F2F7;
  --bg-surface: #FAFAFA;
  --text-primary: #1D1D1F;
  --text-muted: #6B6B70;
  --border-hairline: #E5E5EA;
  --surface-hover: #F7F7F8;
  --radius-container: 10px;
  --space-base: 12px; /* Mobile-first base spacing */
}

/* Tema escuro (exemplo) */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-app: #0F0F10;
    --bg-surface: #121212;
    --text-primary: #EAEAEA;
    --border-hairline: #222225;
  }
}
```

## 2. Tipografia e Ritmo Vertical
- Fonte principal: `Inter`.
- Base: `font-size: 15px` (mobile first).
- H3: 18px; H2: 22px; small/metadata: 13px.
- Vertical rhythm: usar `--space-base` como unidade; margem antes de H2 ≈ `calc(var(--space-base) * 2)` e espaçamento interno `var(--space-base)`.

Exemplo:
```
.section { padding: var(--space-base); }
.section h2 { margin-top: calc(var(--space-base) * 2); font-size: 22px; }
.section h3 { font-size: 18px; }
small, .meta { font-size: 13px; }
```

## 3. Playground
- Layout em grid; itens centralizados (align-items: center; justify-content: center) no cartão/thumbnail.
- Estrutura do item: thumb/icon (centro), H3 (left-aligned), descrição (baixo do título ou ao lado em telas maiores).
- Sub-tabs: `.pg-tab-content { display: none; } .pg-tab-content.active { display: block; }` para prevenir empilhamento de conteúdo.

## 4. Interatividade
- Hover: alterar somente o background para `--surface-hover` ou similar; evitar transform/scale/translate que movam conteúdo.
- Botões de navegação: usar atributo `data-view="<view-name>"` para identificar a view alvo. JS deve trocar views lendo `data-view`, não o texto do botão.

## 5. 'Hire Me' cards
- Devem usar `background: linear-gradient(...)` para se destacar.
- Cartões ainda respeitam `--bg-surface` para conteúdos internos, borda arredondada `--radius-container` e sombra suave.

Exemplo:
```
.hire-card { background: linear-gradient(135deg, #FDE68A 0%, #FBBF24 100%); border-radius: var(--radius-container); box-shadow: 0 6px 18px rgba(29,29,31,0.06); }
```

## 6. Estrutura HTML e Acessibilidade
- DOM deve crescer para conteúdo — não cortar texto com `overflow:hidden` em elementos de conteúdo principal.
- Fornecer labels e roles para controles do Playground (aria-labels, role="tablist"/"tab").

## 7. Traduções e 'Dual Update'
- Processo obrigatório ao alterar conteúdo textual:
  1. Atualizar a string em Português diretamente em `index.html` (conteúdo hardcoded para SEO/first paint).
  2. Atualizar `TRANSLATIONS` em `main.js` para `pt`, `en`, `es` e garantir que a função `applyTranslations()` esteja sincronizada.

- Todos os rótulos e controles técnicos (ex.: 'Resolution', 'Apply', 'Reset') devem existir nas 3 línguas.
- Navegação e lógica devem usar `data-view` e não texto, garantindo que mudanças de idioma não quebrem a navegação.

## 8. Regras de implementação JS
- Usar Vanilla JS; manter módulos pequenos e comentados.
- Inserir um objeto `TRANSLATIONS` no topo de `main.js` com chaves por idioma e por string.
- Função `applyTranslations(lang)` responsável por mapear `data-i18n` attributes para texto:
  - Elementos que precisam ser traduzidos devem ter `data-i18n="key.path"` no HTML.

Exemplo (JS pseudo):
```
// TRANSLATIONS.pt['playground.resolution'] = 'Resolução'
// HTML: <label data-i18n="playground.resolution">Resolução</label>
```

## 9. Checklist de PR / Revisão de Código
- [ ] Texto em Português presente em `index.html` para conteúdo indexável.
- [ ] `TRANSLATIONS` atualizado em `main.js` para pt/en/es.
- [ ] Todos os botões de navegação usam `data-view`.
- [ ] `Hire Me` cards usam `linear-gradient` no background.
- [ ] Variáveis semânticas utilizadas em CSS (nenhum hex direto em componentes).
- [ ] `.pg-tab-content` usado nas sub-tabs do Playground com comportamento `.active`.
- [ ] Hover altera apenas background (sem transformações).

## 10. Observações finais
- O repositório usa Vanilla JS/CSS por escolha. Se for necessário um componente reutilizável mais complexo, criar um pequeno utilitário JS local (ex.: `utils/i18n.js`) — evitar dependências externas.
