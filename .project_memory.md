# Memória do Projeto — curriculo

Esta memória documenta fatos do projeto a serem usados por desenvolvedores e automações.

## Metadados
- Nome do projeto: curriculo
- Pasta principal: `curriculo/`
- Idiomas suportados: `pt` (Português), `en` (Inglês), `es` (Espanhol)

## Design System (fatos)
- Surface Light: `#FAFAFA` (usar variável `--bg-surface`)
- Surface Dark: `#121212` (usar variável `--bg-surface` no tema escuro)
- Background: `#F2F2F7` (usar `--bg-app`)
- Text Primary: `#1D1D1F` (usar `--text-primary`)
- Evitar `#000000` e `#FFFFFF` em componentes de texto e superfícies.
- Content containers: off-white `#FAFAFA`, `border-radius: 10px`, soft shadow.

## Variáveis semânticas recomendadas
- `--bg-app` — fundo da aplicação
- `--bg-surface` — superfícies / cards
- `--text-primary` — texto principal
- `--text-muted` — texto secundário / metadata
- `--border-hairline` — cor da hairline (ex.: `#E5E5EA`)
- `--surface-hover` — cor de hover sutil (ex.: `#FAFAFA` em contexto dark/light quando necessário)

## Tipografia
- Fonte: `Inter` (base mobile-first)
- Tamanho base: `15px`
- H3: `18px`, H2: `22px`
- Metadata / small: `13px`

## Regras de interface e comportamento (resumo)
- Tradução: todo texto visível ao usuário deve existir em PT, EN, ES; controles técnicos (ex.: 'Resolution') também.
- Dual Update obrigatório: textos primários devem estar escritos em Português no `index.html` para SEO/pintura inicial e também presentes em `main.js`/objeto de traduções para sincronização.
- Navegação: usar `data-view` em botões para troca de views, não confiar no texto do botão.
- Playground: layout em grid; itens centralizados vertical e horizontalmente; miniatura/ícone, título H3 (alinhado à esquerda) e descrição; sub-tabs usam `.pg-tab-content` com `display:none` e `.active {display:block}`.
- Hover: feedback por mudança de fundo sutil, sem escala/movimento.
- Hire Me: cards usam `linear-gradient(...)` para fundo.
- Ícones: usar Phosphor Icons (Outline) ou sprite SVG.
- Header: `.site-header` sticky com background sólido e `border-bottom: 1px solid var(--border-hairline)`.
- Vertical Rhythm: espaçamento antes de `H2` ≈ 2x espaçamento dentro da seção.

## Estrutura lógica de views
- Views principais: `resume` e `playground`.
- Playground sub-tabs: `experiments`, `comex-news`, `international-news`.

## Regras de conteúdo
- Entradas do currículo: linguagem objetiva, sem jargões corporativos (no 'synergy', 'holistic', etc.).
- Títulos profissionais: preferir termos técnicos (ex.: 'Desenvolvimento') quando aplicar.
- Dom HTML: expandir para acomodar conteúdo; NÃO truncar visualmente via CSS que esconda texto crítico.

## Integração técnica (fatos práticos)
- Linguagens: Vanilla JavaScript + CSS
- Evitar frameworks UI externos
- Arquivos-chave: `index.html`, `main.js`, `styles.css` (e backup `styles.css.bak`)
- Local onde inserir a memória: `curriculo/.project_memory.md`

## Notas operacionais
- Sempre que for atualizar conteúdo textual para publicar, seguir o processo 'Dual Update':
  1. Atualizar texto em Português em `index.html` (para SEO/initial paint).
  2. Atualizar `TRANSLATIONS` em `main.js` para todas as línguas e acionar função de sincronização.
